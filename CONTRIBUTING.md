# Contributing to Kirov Dynamics

Thank you for your interest! Here is how to get involved.

## Ways to Contribute

### 1. Improve Documentation
- Fix typos, clarify explanations, add examples
- Translate guides into South African languages

### 2. Report Issues
- Open an issue with a clear description
- Include steps to reproduce for bugs
- Suggest improvements with use cases

### 3. Submit Code
- Fork the relevant repository
- Create a branch: `feat/description` or `fix/description`
- Commit with conventional commits (`feat:`, `fix:`, `docs:`, `chore:`)
- Open a Pull Request

### 4. Share Use Cases
- Write a case study for how you use the tools
- Share on LinkedIn and tag @KirovDynamics

## Project Focus Areas

We prioritize contributions that align with our mission — **Practical AI for Africa**:

- Local AI deployment in underserved areas
- Transport intelligence for African cities
- African language NLP and LLM tooling
- Open-source security tooling for African enterprises

## Code of Conduct

Be respectful, constructive, and inclusive. We welcome contributors of all backgrounds.

## Getting Started

Browse the [ecosystem](README.md) and pick a project that interests you. Each repo has its own contributing guide and issue templates.
